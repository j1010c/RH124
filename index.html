1. Accessing the Command Line

🖥️ Virtual Consoles & TTYs

HIGH

Linux provides multiple virtual consoles (TTYs). You can switch between them using Ctrl+Alt+F1 through Ctrl+Alt+F6. The graphical desktop (if present) typically runs on tty1 or tty2.

  • tty1–tty6 = text-based logins (virtual consoles)
  • pts/0, pts/1… = pseudo-terminals (SSH sessions, terminal emulators)
  • Check your current terminal: tty
  • Check logged-in users: w or who
Ctrl+Alt+F2  # Switch to virtual console 2
chvt 3        # Switch to virtual console 3 as root
tty           # Show current terminal device

🔤 Shell Basics & Special Characters

CRITICAL

The default shell in RHEL is bash (Bourne Again Shell). Understanding metacharacters is essential.

  • # – Comment (anything after is ignored)
  • \ – Escape next character / line continuation
  • | – Pipe: send output of one command to another
  • > >> – Redirect output (overwrite / append)
  • 2> – Redirect stderr; &> – redirect stdout+stderr
  • $(cmd) or `cmd` – Command substitution
  • ; – Run commands sequentially
  • && – Run next only if previous succeeded
  • || – Run next only if previous failed
echo "Today is $(date)"   # Command substitution
ls /tmp && echo "success" # Runs echo only if ls succeeds
cat file > output.txt   # Redirect stdout to file
cat file 2> errors.txt  # Redirect stderr to file
2. Managing Files from the Command Line

📁 Linux Filesystem Hierarchy (FHS)

CRITICAL
  • / – Root of the entire filesystem
  • /home – User home directories
  • /root – Root user's home
  • /etc – System-wide configuration files
  • /var – Variable data (logs, spool, databases)
  • /tmp – Temporary files (cleared on reboot)
  • /usr – User programs and data (read-only)
  • /bin → /usr/bin – Common user commands
  • /sbin → /usr/sbin – System admin commands
  • /boot – Boot loader files, kernel
  • /dev – Device files
  • /proc – Virtual filesystem for kernel/process info
  • /sys – Virtual filesystem for hardware/kernel info
  • /mnt / /media – Temporary mount points
💡 Tip: Absolute paths start with /. Relative paths don't. . = current dir, .. = parent dir, ~ = your home.

📄 Essential File Commands

CRITICAL

rm file               # Remove file

🔗 Hard Links vs Soft Links

HIGH
  • Hard link: Another name for the same inode. Works only within same filesystem. If original deleted, link still works.
  • Soft/Symbolic link: A pointer to another file's path. Can cross filesystems. If original deleted, link is broken.
ln file1 hardlink1          # Create hard link
ln -s /path/file1 symlink1  # Create symbolic link
ls -li                      # Show inode numbers (same = hard link)
readlink symlink1            # Show where symlink points

4. Creating, Viewing, and Managing Users and Groups

👤 User Management

CRITICAL

Important files: /etc/passwd (user info), /etc/shadow (encrypted passwords), /etc/group (group info)

useradd john                   # Create user john
useradd -u 1500 -G wheel john  # UID 1500, add to wheel group
useradd -s /sbin/nologin svc   # Service account (no login)
passwd john                    # Set password for john
usermod -aG wheel john         # Add john to wheel group (append)
usermod -L john                # Lock account
usermod -U john                # Unlock account
userdel john                   # Delete user (keep home dir)
userdel -r john               # Delete user AND home dir
id john                        # Show UID/GID for john
chage -l john                  # Show password aging info
chage -M 90 john               # Max 90 days before password expires
⚠️ Exam Alert: Always use -aG with usermod (not just -G) or you will REMOVE the user from all other groups!

5. Controlling Access to Files

🔐 File Permissions (chmod)

CRITICAL

Permissions are shown as rwxrwxrwx → Owner | Group | Others

  • r = read (4)    w = write (2)    x = execute (1)
  • rwx = 7   rw- = 6   r-x = 5   r-- = 4
chmod 755 file    # rwxr-xr-x (owner=7, group=5, others=5)
chmod 644 file    # rw-r--r-- (typical file)
chmod u+x file   # Add execute for owner
chmod go-w file  # Remove write from group and others
chmod a=r file   # Set all (a) to read only
chown john file  # Change owner to john
chown john:devs file  # Change owner and group
chgrp devs file  # Change group only
umask            # Show default permission mask
umask 027        # New files get 640, dirs get 750

🏷️ Special Permissions: SUID, SGID, Sticky Bit

HIGH
  • SUID (4): File runs as the owner's UID. -rwsr-xr-x (s in owner execute)
  • SGID (2): File runs as group's GID, OR on directory: new files inherit group. -rwxr-sr-x
  • Sticky Bit (1): On directory: only owner can delete their own files. Used on /tmp. drwxrwxrwt
chmod 4755 /usr/bin/prog   # Set SUID
chmod 2755 /shared          # Set SGID on directory
chmod 1777 /tmp             # Sticky bit (like /tmp)
chmod u+s file              # Set SUID (symbolic)
chmod g+s dir               # Set SGID (symbolic)
chmod o+t dir               # Set sticky bit (symbolic)

7. Controlling Services and Daemons (systemd)

🔧 systemctl – Service Management

CRITICAL 
systemctl start httpd       # Start service
systemctl stop httpd        # Stop service
systemctl restart httpd     # Stop then start
systemctl reload httpd      # Reload config without stopping
systemctl enable httpd      # Enable at boot
systemctl disable httpd     # Disable at boot
systemctl enable --now httpd # Enable AND start NOW
systemctl status httpd      # Show status + recent logs
systemctl is-active httpd   # Returns "active" or "inactive"
systemctl is-enabled httpd  # Returns "enabled" or "disabled"
systemctl list-units        # List all active units
systemctl list-unit-files   # List all unit files + enabled state

# System targets (run levels equivalent)
systemctl get-default             # Show current default target
systemctl set-default multi-user  # Boot to CLI
systemctl set-default graphical   # Boot to GUI
systemctl isolate rescue.target   # Switch to rescue mode NOW
💡 Always use enable –now on the exam — it starts the service immediately AND enables it at boot in one command.
8. Configuring and Securing SSH

🔒 SSH Configuration & Key-Based Auth

HIGH 
ssh user@hostname         # Connect to remote host
ssh -p 2222 user@host     # Connect on port 2222
ssh-keygen                # Generate key pair (creates ~/.ssh/id_rsa)
ssh-copy-id user@host     # Copy public key to remote host
ssh-keygen -t ed25519    # Generate ED25519 key (recommended)

# Key files
~/.ssh/id_rsa              # Private key (keep SECRET: chmod 600)
~/.ssh/id_rsa.pub          # Public key (share this)
~/.ssh/authorized_keys     # Remote: public keys allowed to login
~/.ssh/known_hosts         # Hosts you’ve connected to

# sshd config: /etc/ssh/sshd_config
PermitRootLogin no        # Disable root SSH login
PasswordAuthentication no # Force key-based auth only
Port 2222                  # Change SSH port
⚠️ After editing /etc/ssh/sshd_config: always run systemctl restart sshd and verify with systemctl status sshd

14. Archiving and Transferring Files

🗜️ tar, gzip, and File Transfer

HIGH 
# tar — archive tool (remember: czf = Create Zip File)
tar czf archive.tar.gz /dir/    # Create compressed archive
tar cjf archive.tar.bz2 /dir/   # bzip2 compression
tar cJf archive.tar.xz /dir/    # xz compression
tar xzf archive.tar.gz          # Extract gzip archive
tar xf archive.tar.gz -C /dest  # Extract to specific dir
tar tzf archive.tar.gz          # List contents without extracting
tar tf archive.tar              # List uncompressed archive

# File transfer
scp file.txt user@host:/dest/    # Secure copy to remote
scp -r dir/ user@host:/dest/    # Copy directory to remote
rsync -avz src/ user@host:/dst/  # Sync files (efficient)
💡 Memory trick: c=Create, x=eXtract, t=lisT, f=File, z=gZip, j=bzip2, J=xz, v=Verbose

16. Bash Shell Scripting Basics

📜 Shell Script Fundamentals

MEDIUM 
#!/bin/bash
# Always start with shebang line

# Variables
NAME=“Alice”
echo “Hello $NAME”

# If statement
if [ ”$NAME” == “Alice” ]; then
echo “Found Alice”
elif [ ”$NAME” == “Bob” ]; then
echo “Found Bob”
else
echo “Unknown”
fi

# For loop
for i in 1 2 3; do
echo “Number: $i”
done

# While loop
COUNT=0
while [ $COUNT -lt 5 ]; do
echo $COUNT
COUNT=$((COUNT+1))
done

# Test operators: -eq -ne -lt -gt -le -ge (numbers)
#                 == != (strings)
#                 -f (file exists) -d (dir exists) -z (empty string)

# Make executable and run:
chmod +x script.sh && ./script.sh
